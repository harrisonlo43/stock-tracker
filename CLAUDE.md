# 給 Claude 的操作說明（股票觀察清單管理）

這個 repo 是「台股產業趨勢觀察」專案。使用者可能在**手機上用 Claude 開一個雲端 session**，然後直接下中文指令要你管理觀察清單。請依下列規則操作，全程使用繁體中文。

## 觀察清單檔案：`watchlist.txt`

- 分類註解以 `#` 開頭。
- 每個股票代碼獨立一行，代碼後用 `# 中文名稱` 標註。
- 目前分類：
  - **半導體個股**
  - **大盤槓桿 ETF（正2）**
  - **主動式台股 ETF**
  - （未來可新增其他分類，例如 金融、傳產、航運、生技、AI 供應鏈 等）

## 常見指令與對應動作

### 1. 新增代碼（例如：「新增 2454」「加入 2317 鴻海」）
1. 用 WebSearch 查該代碼的**正式名稱**與**所屬產業/分類**。
2. 在 `watchlist.txt` 對應分類區塊下新增一行 `代碼  # 中文名稱`；若沒有合適分類就新增一個 `# === 分類名 ===` 區塊。
3. 若代碼已存在，回覆使用者「已在清單中」，不要重複新增。
4. commit + push（見下方 Git 設定），並用一兩句白話回覆使用者：新增了什麼、屬於哪個產業。

### 2. 移除代碼（例如：「移除 00685L」「把台積電拿掉」）
1. 從 `watchlist.txt` 刪掉該行（可用中文名稱或代碼比對）。
2. 若刪除後某分類區塊沒有任何代碼，也把該空區塊標題一併移除。
3. commit + push，並回覆使用者已移除。

### 3. 查看清單（例如：「現在清單有哪些」）
- 讀 `watchlist.txt`，用分類條列方式回覆，不需 commit。

### 4. 重新分類（例如：「幫我重新整理分類」）
- 依產業重新分組 `watchlist.txt`，保持格式，commit + push。

```
git config user.name "harrisonlo43"

git add -A
git commit -m "更新觀察清單：<簡述變更>"

```

## 白話原則

回覆使用者時，避免艱澀術語。像「正2 ETF」要順帶說明「追蹤大盤兩倍、漲跌都放大、波動較大」；「主動式 ETF」要說明「由基金經理人主動選股，不是照指數」。目標是讓非專業的人也一看就懂。
